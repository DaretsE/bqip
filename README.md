# BQDiptv — Android TV IPTV-плеер

Нативное Android TV приложение на Kotlin + Jetpack Compose, собранное на основе
прототипа `BQDiptv-ui-redesign.html` (дизайн, цвета, анимации панелей,
логика управления). Собирается автоматически на GitHub Actions в APK,
который сам умеет проверять и устанавливать обновления через GitHub Releases.

## Что уже реализовано

- Воспроизведение M3U-плейлистов через ExoPlayer/Media3 (HLS/DASH/прогрессив)
- Разбор EPG (XMLTV)
- Первичная настройка "с телефона" — TV поднимает локальный веб-сервер и
  показывает QR-код, телефон открывает страницу и вводит ссылку на плейлист
- Панели: левое меню категорий, браузер каналов с превью, EPG, поиск по
  программе передач, настройки
- Автопереключение на следующий канал при ошибке воспроизведения
- Избранное
- **Автообновление**: приложение раз в сутки (фоново, через WorkManager) и
  по кнопке в Настройках опрашивает `GET /repos/{owner}/{repo}/releases/latest`,
  сравнивает версию, скачивает APK и предлагает установить — без Google Play
- Анимации панелей (выезд/затухание), цвета и длительности перенесены из
  CSS прототипа с точными `cubic-bezier`, а не "на глаз" — см. `ui/theme/Motion.kt`
- Адаптация под слабые ТВ-приставки: авто-детект (RAM/API) + ручной тумблер
  "Экономичный режим" в Настройках, отключающий дорогие эффекты

### Осознанно упрощено (см. комментарии в коде)

- `backdrop-filter: blur()` прототипа (блюр видео позади панелей) не
  воспроизведён буквально — технически ExoPlayer рисует видео через
  `SurfaceView` в обход обычного дерева отрисовки Compose, так что захватить
  кадр для блюра нельзя без переключения на `TextureView` (дороже по GPU —
  ровно то, чего мы избегаем ради слабых устройств). Панели используют
  адаптивную полупрозрачность + лёгкий блик вместо реального блюра.
  Подробности и путь к "настоящему" блюру — в `ui/theme/Glass.kt`.
- D-pad навигация сейчас на уровне "зон" (вверх/вниз — канал, влево/вправо —
  панели), а не поэлементный фокус по каждой строке списка.
- Не реализованы: скринсейвер, модальные диалоги, сворачивание левого меню
  в узкую иконку-рейл при просмотре списка каналов.

## Структура проекта

```
app/src/main/java/com/bqdiptv/tv/
  model/       — Channel, EpgProgram
  data/        — M3uParser, XmltvParser, SettingsRepository (DataStore)
  player/      — обёртка над ExoPlayer
  setup/       — локальный HTTP-сервер для настройки с телефона + QR
  update/      — UpdateManager (GitHub Releases), фоновая проверка
  ui/          — Compose-экраны, анимации, тема
.github/workflows/
  ci.yml       — сборка debug-APK на каждый push (для проверки, что всё компилируется)
  release.yml  — сборка + публикация release-APK по тегу vX.Y.Z
```

## Как запустить сборку у себя

### 1. Создайте репозиторий на GitHub

Загрузите содержимое этой папки в новый репозиторий, например
`github.com/ВАШ_ЛОГИН/BQDiptv`.

### 2. Впишите имя репозитория в приложение

В файле `app/build.gradle.kts` найдите строку:

```kotlin
buildConfigField("String", "GITHUB_REPO", "\"YOUR_GITHUB_USERNAME/BQDiptv\"")
```

и замените на ваш `owner/repo`, например `"ivanov/BQDiptv"`. Это единственное
место, которое нужно поменять вручную — именно сюда приложение будет
стучаться за обновлениями.

### 3. Проверьте, что всё компилируется

Запушьте любой коммит — сработает `ci.yml` и соберёт debug-APK
(Actions → выбрать ран → Artifacts → `bqdiptv-debug`). Это не релиз, просто
проверка сборки.

### 4. Выпустите первый релиз

```bash
git tag v1.0.0
git push origin v1.0.0
```

Сработает `release.yml`: соберёт release-APK, посчитает `versionCode` из
тега (1.0.0 → 10000, по формуле `MAJOR*10000+MINOR*100+PATCH`), опубликует
релиз на GitHub с прикреплённым APK. Установите этот APK на телевизор
вручную один раз (через USB/файловый менеджер/ADB) — дальше обновления
будут приходить сами.

### 5. Проверка автообновления

Поднимите версию, например `v1.0.1`, и запушьте тег так же, как выше.
На следующей фоновой проверке (раз в сутки) или по кнопке
"Проверить обновления" в Настройках приложение увидит новую версию,
скачает APK и предложит установить. При первом обновлении Android
попросит разрешение "Установка неизвестных приложений" для этого
приложения — это нормально, дальше оно не спросит повторно.

## Подпись APK (опционально, для продакшна)

Сейчас `release.yml` собирает APK с debug-подписью — этого достаточно, чтобы
всё работало, включая автообновления. Чтобы перейти на "настоящую"
release-подпись (рекомендуется перед реальным распространением):

1. Сгенерируйте keystore:
   ```bash
   keytool -genkey -v -keystore release.keystore -alias bqdiptv \
     -keyalg RSA -keysize 2048 -validity 10000
   ```
2. Закодируйте его в base64: `base64 -w0 release.keystore > release.keystore.b64`
3. В репозитории: Settings → Secrets and variables → Actions → добавьте:
   - `KEYSTORE_BASE64` — содержимое `release.keystore.b64`
   - `KEYSTORE_PASSWORD`, `KEY_ALIAS`, `KEY_PASSWORD`
4. Готово — `release.yml` сам обнаружит секреты и подпишет release-ключом.

**Важно:** если вы переключаетесь с debug- на release-подпись на устройствах,
где уже установлена debug-версия, Android откажется ставить обновление
поверх (разные подписи) — нужно будет один раз переустановить вручную.
Дальше все обновления с release-подписью будут ставиться поверх друг друга
без проблем.

## Формат плейлиста и EPG

- Плейлист — стандартный M3U/M3U8 (`#EXTINF` с `tvg-id`, `tvg-logo`,
  `group-title`)
- EPG — стандартный XMLTV (`<programme channel="" start="" stop="">`)

`tvg-id` в плейлисте должен совпадать с атрибутом `channel` в XMLTV, иначе
программа передач для канала не найдётся.

## Известные ограничения окружения сборки

Проект собирается через `gradle` (не через закоммиченный `gradlew`) —
`.github/workflows/*.yml` используют `gradle/actions/setup-gradle`, который
сам ставит нужную версию Gradle на раннере. Для локальной сборки в Android
Studio откройте папку проекта как обычно — Android Studio сама сгенерирует
wrapper при первом открытии.
